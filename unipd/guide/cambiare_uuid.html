Come cambiare l'UUID dopo aver modificato le partizioni di un sistema Linux

1. Problema

E' necessario dopo aver modificato la "posizione" (allocazione di memoria) di una o più partizioni nel proprio filesystem dover modificare le proprie impostazioni di configurazione, per rendere più efficace e veloce il nostro sistema linux all'avvio. Non prendere provvedimenti può portare a un rallentamento significativo in fase di boot, poichè il sistema non riesce a identificare a dovere le partizioni da utilizzare.

2. Soluzione

La soluzione è piuttosto semplice. Seguire i seguenti passi

2.1. Entrare in modalità recovery (recovery mode)

All'avvio, nel momento in cui il sistema deve ancora entrare in fase di boot selezioniamo l'opzione "Advanced" per linux. Così facendo si aprirà un menù. Selezionare una delle opzioni contenenti recovery o GRUB.

2.2. Visualizzare gli UUID delle nuove partizioni

Per visualizzare i nuovi UUID da inserire poi nella configurazione, eseguire il comando blkid o in alternativa (per una visualizzazione più chiara) lsblk -f 

lsblk -f

Una volta visualizzati gli UUID, copiarli.

2.3. Modifica file di configurazione

Entrare nella cartella /etc/ e modificare il file fstab. In questo file sono espresse le partizioni che devono essere utilizzate, con i rispettivi ruoli. 

sudo nano /etc/fstab

Aprire il file con un editor di testo (come nano) e sostituire i vecchi UUID con i nuovi.

Salvare poi con C-O e chiudere l'editor con C-X (nel caso sia stato utilizzato nano).

2.4. Boot

Una volta configurato correttamente basterà premere C-D e continuare con il boot del dispositivo. Le volte successive sarà sufficiente avviare il proprio sistema normalmente.
